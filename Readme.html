Building JBarcodeBean

A Makefile is included which will build JBarcodeBean on GNU/Linux systems.

Assuming you've got Java 2 Standard Edition development kit installed, and the PATH environment variable is correctly set, you can build everything by just typing make in the jbarcodebean directory.

Running the Demo

JBarcodeBean is supplied with a demo application, packaged as an Executable JAR file, Demo.jar.

The demo allows you to experiment with the bean's properties, test the output (in GIF format or hardcopy), and view the documentation.

To run the demo application, copy Demo.jar and JBarcodeBean.jar to a directory on your computer, then launch Demo.jar as an Executable Jar. The command line invocation is:

java -jar Demo.jar

JBarcodeBean requires a JavaTM 2 runtime environment (version 1.2.2 or later).

A JavaTM 2 runtime environment is included in the JavaTM 2 SDK, and is also available separately. Click here to download Sun's JavaTM 2 SDK or JavaTM 2 Runtime Environment

Installing in an IDE

Being a JavaBeansTM component, JBarcodeBean integrates seamlessly into the component palette of JavaTM 2 compatible GUI-builders. Since it is written entirely in pure JavaTM, it will work on any JavaTM 2 platform (version 1.2.2 or later).

Consult the documention of your IDE for instructions on adding JavaBeans to the tool's component palette. Typically, this involves copying the bean file (JBarcodeBean.jar) to a specific directory where the tool looks for bean components, or adding a reference to the bean file in a configuration dialog.
